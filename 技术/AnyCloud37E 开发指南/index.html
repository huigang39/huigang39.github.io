# AnyCloud37E 开发指南

# 烧录

编译完之后， auto_build.sh 这个脚本会自动把生成的镜像复制到 tools/burntool 文件夹，但是这里面的烧录工具是 .exe 文件格式，也就是说只能在 Windows 下运行，需要把这个文件夹复制到 Windows 系统下，然后打开。

打开之后会弹出 Cannot find file: cloudOS.dtb ，不用管，点击确定进入就行了。进来之后，顶栏 Setting 里的 Language 可以修改为中文，然后点击配置文件，选择 config_64M_spinor.txt 即可。

然后点击下面的那个设置（大按钮那个），弹出来让你选生产者还是研发者，这个无所谓（研发者可配置的选项多一些），密码是统一的 anyka 。进入之后选择 burn ，烧录设置 couldOS.dtb 这一项需要修改成存在且正确（就是这个文件名必须在这个烧录工具的目录下）的文件名，例如你可以改为 EVB_CBDR_AK3760E_V1.0.1.dtb ，这个配置文件需要根据板子的具体型号来决定，不确定的就多烧录几次尝试一下。

配置完成之后，插上 USB 烧录端口（注意眺帽位置，按平台开发板使用说明书操作），长按 Boot 按键，不要松开，然后按下电源按键，软件应该会开始自动烧录，当软件有反应的时候，立刻松开 Boot 按键，等待它烧录完成即可。

# 调试

烧录完系统之后，将 USB 数据线（注意：必须要能传输数据的线，不要拿只能充电的残血 USB）插入板子的接口（同样注意跳帽位置），打开任意一款串口工具（Windows 推荐 MobaXterm，Linux 推荐 Minicom），按下板子的 Reset 重置按键。串口工具应该就会滚动显示开发板系统一连串的启动信息。

等待终端让你输入账户，输入 root 并回车，密码没有，直接回车就行了。可能这个时候会有个叫 udhcpc 的程序一直提示一些信息，干扰你的输入，这时你就可以用迅雷不及掩耳之势输入 killall udhcpc 杀死这个进程即可。

接下来就是正常的 Linux 操作环节了，没什么好说的。需要注意的是，这系统并不是什么 Ubuntu 或者其他发行版，更没有 apt、dnf、yum、pacman 等等包管理工具，这只是一个移植的 Linux，里面有什么软件工具取决于你烧录进去的有没有这些东西，这个里面只带了 BusyBox，所以 BusyBox 里面自带的几百个工具都能使用，例如 telnet、mount 等等工具。
