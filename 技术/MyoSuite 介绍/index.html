# MyoSuite 介绍

# 概述

MyoSuite 是由 Meta（原 Facebook）开发的一款用于肌肉骨骼运动控制的富接触模拟套件，使用 MuJoCo 物理引擎模拟并包装在 OpenAI API 中，以实现机器学习在生物力学控制问题中的应用。Meta 将其称之为一个将神经和运动智能统一起来的具象人工智能平台。

以下内容节选自官方博客：

我们已经开发了一个新的具身（embodied）[1]人工智能平台，称为 MyoSuite，通过统一运动和神经智能，将机器学习（ML）应用于生物力学控制问题。

MyoSuite 中的肌肉骨骼模型在满足现代 ML 算法的数据要求方面比其他模拟器快 4000 倍。

这项研究可能会对开发假肢和伤后康复等领域产生重大影响，因为我们可以完全详细地生成生理学上的真实动作，如旋转笔或操纵保龄球。

当我们为元宇宙进行建设时，这些模型将帮助开发者建立动作更真实的化身，使体验更具表现力和沉浸感。

# 使用

# 安装

依赖软件：Git、Miniconda（建议）

步骤（我自己在 Linux 系统下的配置，Windows 系统需要自行查阅一下官方文档）：

  1. sudo apt-get install libgl1-mesa-dev libgl1-mesa-glx libglew-dev libosmesa6-dev ，否则后面运行时会报错，错误为 fatal error: GL/osmesa.h: No such file or directory 。

  2. sudo apt-get install patchelf ，否则后面运行时会报错，错误为 FileNotFoundError: [Errno 2] No such file or directory: 'patchelf': 'patchelf' 。

  3. 创建虚拟环境： conda create --name MyoSuite python=3.7.1

  4. 激活虚拟环境： conda activate MyoSuite

  5. 安装库文件： pip install -U myosuite

  6. 克隆官方源代码仓库： git clone --recursive https://github.com/facebookresearch/myosuite.git ，如果出现 timeout 或者 http/https 错误就说明需要翻墙了。

  7. 安装 Python 依赖库： cd myosuite && pip install -e -r requirements

  8. （可选）测试安装： python myosuite/tests/test_myo.py ；随机控件可视化环境： python myosuite/utils/visualize_env.py --env_name myoElbowPose1D6MRandom-v0 。

  9. 如果出现其他错误，请查阅 Know Issues。

# 软件介绍

# 模型

MyoSuite 由三种模型组成：myoFinger，myoElbow，myoHand。

# myoFinger

简化和直观的模型的 4 自由度（DoF）手指，其通过一系列 5 个简化的拮抗肌腱单元驱动。还为其机器人提供简单的扭矩执行器，以方便比较研究。

# myoElbow

一个 2DoF 人类肘关节的模型：基于 OpenSim 的默认测试手臂模型，并使用多个激动剂 / 拮抗剂对（3 个屈体和 3 个伸体）进行驱动。

灵巧的人手需要协调多种高度冗余的肌肉，这些肌肉对各种关节有互补和拮抗作用。这个肌肉骨骼模型由 29 块骨头、23 个关节和 39 个肌肉 - 肌腱单元组成。

# 任务

# 手指关节姿态

# 目标

  • 摆出一个关节姿态

# 变种

  • 简易：移动到一个固定的指定关节姿态 myoFingerPoseFixed-v0

  • 困难：移动到随机选择的关节姿态 myoFingerPoseRandom-v0

  • 更多：随机重设、无重设、密集奖励、稀疏奖励（强化学习中的专有名词）

# 指尖伸展

# 目标

  • 用指尖触及

# 变种

  • 简易：触及一个固定的位置 myoFingerReachFixed-v0
  • 困难：触及随机位置 myoFingerReachRandom-v0
  • 更多：随机重设、无重设、密集奖励、稀疏奖励（强化学习中的专有名词）

# 肘关节姿态

具有 6 块肌肉（3 个屈肌和 3 个伸肌）的 myoElbow 模型被简化为只有肘部旋转。虽然它不是一个生理上准确的模型，但它可以是一个非常简单的模型，用于对初始控制方案进行故障排除。

# 目标

将肘部移动到指定的姿态。

# 变种

  • 简易：使用 1 块肌肉移动到随机关节姿态 ElbowPose1D1MRandom-v0
  • 困难：使用 6 块肌肉移动到随机关节姿态 myoElbowPose1D6MRandom-v0
  • 更多：随机重设、无重设、密集奖励、稀疏奖励（强化学习中的专有名词）

# 手关节姿态

驱动 myoHand 模型以实现关节姿态。除了进行协调的运动外，避免自碰撞在解决此任务方面也带来了额外的挑战。

# 目标

  • 摆出手部姿态

# 变种

  • 简易：移动到固定关节姿态 myoHandPoseFixed-v0
  • 困难：移动到随机选择的关节姿态 myoHandPoseRandom-v0
  • 更多：随机重设、无重设、密集奖励、稀疏奖励（强化学习中的专有名词）

# 手尖伸展

使用 myoHand 模型进行伸展运动，让指尖达到特定的空间位置。除了进行协调的运动外，避免自碰撞在解决此任务方面也带来了额外的挑战。

# 目标

  • 使用指尖伸展

# 变种

  • 简易：使指尖到达固定位置 myoHandReachFixed-v0
  • 困难：使指尖到达随机位置 myoHandReachRandom-v0
  • 更多：随机重设、无重设、密集奖励、稀疏奖励（强化学习中的专有名词）

# 手钥匙转动

使用 myoHand 模型旋转钥匙。此任务的复杂性是由于仅使用食指和拇指手指来解决与按键间歇性接触的任务。

# 目标

协调手指移动以旋转钥匙。

# 变种

  • 简易：实现钥匙的半旋转 myoHandKeyTurnFixed-v0
  • 困难：实现全旋转 + 随机初始配置 myoHandKeyTurnRandom-v0
  • 更多：随机重设、无重设、密集奖励、稀疏奖励（强化学习中的专有名词）

# 手持对象保持

myoHand 模型将手中的对象移动到给定的方向，而不会将其放下。此任务的复杂性是由于物体和多个手指之间的间歇性接触需要协调以稳定物体。

# 目标

重新定位对象以到达给定目标，而不会丢弃它。

# 变种

  • 简易：重新定位到固定位置 myoHandObjHoldFixed-v0
  • 困难：将随机对象重新定位到随机位置 myoHandObjHoldRandom-v0
  • 更多：随机重设、无重设、密集奖励、稀疏奖励（强化学习中的专有名词）

# 手笔旋转

myoHand 模型将手中的笔旋转到给定的方向，而不会掉落。此任务的复杂性是由于在尝试稳定对象时对象与多个手指之间的间歇性接触。

# 目的

旋转对象以达到给定方向（由场景中的绿色对象指示），而不会将其放下。

# 变种

  • 简易：旋转到固定方向 myoHandPenTwirlFixed-v0
  • 困难：随机方向旋转 myoHandPenTwirlRandom-v0
  • 更多：随机重设、无重设、密集奖励、稀疏奖励（强化学习中的专有名词）

# 手保定球

涉及两个自由浮动的球体在手掌上同时旋转的 myoHand 模型。这项任务既需要灵活性，也需要协调性。

# 目标

实现球围绕彼此的相对旋转，而不会掉落它们。

# 动作空间

整套肌肉 [0,1][0, 1][0,1]

# 观察空间

所有关节角度 [−,ππ][-, \pi \pi][−,ππ]

# 变种

  • 简易：交换球的位置 myoHandBaodingFixed-v1
  • 困难：实现连续旋转 myoHandBaodingRandom-v1
  • 更多：稀疏奖励，3 种不同的密集奖励选项可供选择

# 非平稳性任务变化

# 肌肉减少症

肌肉减少症是一种肌肉疾病，通常发生在老年人群中（Cruz-Jentoft 和 Sayer（2019）），其特征在于肌肉质量或体积的减少。从 20 岁到 40 岁，握力的峰值可以降低到 50%（Dodds 等人（2016））。我们将每块肌肉的肌肉减少症建模为其最大等距力的 50%。

# 疲劳

肌肉疲劳是一种短期（几秒到几分钟）效应，发生在持续或重复的自主运动之后，它与创伤有关，例如累积创伤障碍（Chaffin 等人（2006））。动态肌肉疲劳模型（Ma 等人（2009））被整合到建模框架中。该模型基于不同类型的肌肉纤维具有不同贡献和抗疲劳性的想法（Vøllestad（1997））。当前的实现经过简化，以考虑所有肌肉的相同疲劳因子，并且该肌肉可能完全疲劳。

# 肌腱转移

与所有肌肉发生的肌肉疲劳或肌肉减少症相反，肌腱移植手术可以针对单个肌腱单位。肌腱移植手术允许将肌肉力量的应用点从一个关节 DoF 重定向到另一个关节 DoF（见下文）。它可用于在受伤后重新获得关节或肢体运动的功能控制。目前手中的一个程序是伸肌的 ** 肌腱转移（EIP）** 以取代伸肌 Pollicis Longus（EPL）（Gelb（1995））。EPL 破裂可能发生在手腕骨折后，并导致拇指伸展失去控制。我们介绍了一种物理肌腱转移，其中肌腱的 EIP 应用点从索引移动到拇指，并且 EPL 被移除。

# 外骨骼辅助

外骨骼辅助康复由于其多重益处而变得越来越普遍（Jezernik 等人（2003））（Nam 等人（2017））。肘部外骨骼的建模是通过理想的致动器完成的，并增加了两个支撑，上臂的重量为 0.101 Kg，前臂的重量为 0.111 Kg。外骨骼给予的帮助是生物关节扭矩的百分比，这是基于 Durandau 等人（2019）中提出的神经肌肉骨骼控制器。
